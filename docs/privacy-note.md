# WorkTrace 员工隐私说明

这份说明是写给使用 WorkTrace 的员工看的，重点只有一句话：

WorkTrace 的目标是帮你回顾你自己的工作事项，不是偷偷收集你的个人隐私。

## 1. 它会做什么

- 读取你在目标日期里发过消息或做过 reaction 的飞书工作会话
- 提取和你自己直接相关的工作事件
- 在你本地生成一份 Markdown 文件
- 默认通过飞书机器人先把这份文件发给你自己

## 2. 它默认不会做什么

- 不会默认直接发给领导
- 不会默认把所有聊天长期保存下来
- 不会在最终 Markdown 里显示群名、open_id、消息 ID、会话 ID 或参与人名单
- 事件正文只在责任分工、任务指派、确认沟通对象等确有必要时保留姓名
- 不会把私人聊天当成项目目标去抓取

## 3. 你需要知道的真实边界

- WorkTrace 会通过你本机上的 `lark-cli` 读取聊天
- WorkTrace 会把必要的精简文本上下文发送到你配置的在线模型服务
- 图片摘要启用时，符合配置限制的聊天图片会临时下载并发送到同一在线模型服务
- 模型明确请求时，指定文本附件或飞书文档正文也可能发送到在线模型服务
- 所以你应当确认 `.env` 里填写的模型地址和密钥，是你自己认可的配置

这意味着它不是“绝对不外发任何内容”的纯本地方案。  
如果你不认可某个模型服务，就不要使用那套配置。

## 4. 当前项目怎么尽量减少隐私暴露

- 只处理和你自己直接相关的工作事项
- 最终产出只保留日期、事件标题、内容、具体对象、保留理由、保留依据和涉及文件
- 默认强制 `/no_think`
- 请求体显式关闭推理过程
- 文档链接主要保留给你自己后续回忆细节，不作为模型复杂推理的输入重点
- 你可以通过 `config/conversation_blacklist.json` 在消息采集前彻底排除指定会话
- 正式运行默认不保存原始聊天；`--debug-output` 会保存裁剪后的上下文和模型输出，只应临时开启

## 5. 你有自主权

- 你会先收到由飞书机器人发给你自己的 Markdown
- 你可以先审阅、修改，再决定是否转发
- 如果你对配置或隐私边界有疑虑，应先停下来确认，而不是直接运行
